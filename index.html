Word Trees

live site
code

This project visualizes the tree-like relationship formed by a word and its synonyms. I built it out of interest in exploring relationships between similar words. A tree of synonyms is built out from the root word in a breadth-first manner. The tree is then searched for the target word if one is provided.

Take "fun" and "gaiety" for example. A path from "fun" to "gaiety" exists because "merriment" is a synonym of "fun", and "gaiety" is a synonym of "merriment".

The synonym data from this project comes from Wordnet, a large lexical database of English words that groups nouns, verbs, adjectives, and adverbs into sets of synonyms. This corpus for Wordnet is used to harness the data. It's provided by NLTK (Natural Language Toolkit), a really cool, extensive Python toolkit for working with language data. On the frontend, d3 is used to render the tree.

Slap

live site
code

This is a clone of the messaging app Slack. I made it from scratch, so I'm pretty proud of it. It's got a Rails backend and a React/Redux frontend. Building this project really showed me the power of Rails associations, Jbuilder, Redux, and building reusable React components. I'm really happy with how my DMs are implemented. When you 'remove' a DM, if you try to create one with the same people, rather than creating an entirely new DM, that existing DM and its messages will be fetched from the database (just like Slack :)) I also implemented debouncing, so that when searching for users or channels, the search query will fire off only once the user stops typing for .5 seconds.
